# Notes
## Summary
>[!Definition]
>An individual’s computational phenotype is defined as a set of mechanistically interpretable parameters obtained from fitting computational models to behavioral data.

>[!Important]
>the interpretation of these parameters hinges critically on their
psychometric properties,  which are rarely studied.

#### Results
> many  phenotype  dimensions  covary  with  practice  and  affective  factor^[seems imminently unsurprising, but I guess good to confirm...]


>[! Takeaway]
>...what appears to be unreliability may reflect previously unmeasured structure.

- we fit two statistical models to the data,  an “independent” and a “dynamic” model

**Independent Model**
- the parameters for each participant were assumed to be drawn independently each week from a participant-specific distribution, which was specified by a set of free parameters.

**Dynamic Model**
- formalizes a more structured set of assumptions about how the computational phenotype evolves over time, thus allowing us to make insightful inferences about sources of its temporal variability.^[What does this mean?]
